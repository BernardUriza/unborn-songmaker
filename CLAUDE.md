# unborn-songmaker — project memory

A rules-based generative sound + music engine. Plain-words / JSON spec in, `.mp3`
out. Homage to **Chris Korda** (Church of Euthanasia; the Polymeter MIDI
Sequencer): music emerges from a readable rule network, never a neural black box.

## The cardinal design law: everything lives in Korda's contract

The engine has TWO layers and the boundary is sacred:

- **Contract layer (Korda, pure):** `Track` + `Modulation` + steps. A Track is a
  loop with its own `length` (in steps) and `quant` (ticks/step); relatively-prime
  lengths slip into polymeter. Modulations let one track reshape another every
  step (`note`, `velocity`, `mute`, `position`). This is `track.py` / `sequencer.py`
  and it is a clean-room port of Korda's `TrackDef.h` (GPL algorithm, not code).
- **Render layer (the synth Korda leaves out):** voices, FX, reverb, sidechain,
  mp3 export. Korda emits MIDI to external synths; this is the synthesizer below
  the contract. `synth.py`, `drums.py`, `soundbank/`, `fx.py`, `render.py`.

**When adding ANY capability, ask first: does it fit the contract or the render
layer?** Drums/bass/voice = Tracks with a different `voice` (contract untouched).
Swing = a native Track property (`Track.swing`, restored from TrackDef.h).
Ducking = a velocity/mute Modulation or render-layer sidechain. Filter movement =
a CONTROL-track modulation. Arrangement = `Track.enter`/`exit` (bar windows).
Melody = a modulator track feeding `MOD_NOTE`. Never bolt on a parallel system
(Art. 6) — express it through the contract; only true audio synthesis is render.

## Files

- `cli.py` — `python cli.py cue <name>|all` (one-shot SFX) · `python cli.py sculpture specs/<x>.json`
- `unborn/track.py` — `Track` (name,type,note,length,quant,offset,swing,velocity,voice,steps,mute,fx,enter,exit), `Modulation`
- `unborn/sequencer.py` — steps tracks, applies modulations, emits `NoteEvent`s
- `unborn/synth.py` — additive voices (bell=inharmonic metal, harmonic, pad, subbass) + `midi_to_freq`
- `unborn/drums.py` — kick/hat/clap/bass (render-layer DSP)
- `unborn/soundbank/` — auto-registering sound library (see below)
- `unborn/fx.py` — per-voice FX chain: reverse, pitch, granular, ring, crush, downsample, drive, bandpass
- `unborn/render.py` — mix, VOICE_GAIN staging, sidechain duck, reverb, mp3 via ffmpeg
- `unborn/spec.py` — JSON spec -> Sequencer; `euclid()` rhythm helper
- `specs/*.json` — the tracks (the catalog)
- `samples/` — recorded voice (gitignored; local/private)

## The soundbank — 70 Codex-generated voices, auto-registered

`soundbank/__init__.py` scans every `soundbank/*.py` and collects each public
`def name(freq, dur) -> np.ndarray` into `SOUNDBANK`, merged into the voice
registry. **Generated by Codex** (`codex exec`, the FI motor) in 4 parallel
batches, **each validated here before trust** (70/70 render, no NaN):
`drums_house.py` (18), `tonal.py` (18: rhodes/marimba/plucks/leads), `nature.py`
(17: water/fire/wind/leaves), `city.py` (17: sirens/cars/crowd/helicopter).
To add more: fan out a `codex exec` per module file, validate each (renders,
non-silent, no NaN, distinct), drop into `soundbank/` — zero manual wiring.

## Spec grammar (the LLM-driven glass box)

Top: `name, tempo, ticks_per_beat, bars, beats_per_bar, sidechain{source_voice,
amount,release}, reverb{amount,decay}`. Each track: `voice` (any registered, or
`sample:<name>`), `note`, `quant`, `length`, `steps[]` or `euclid{pulses,length,
velocity}`, `swing`, `offset`, `enter`/`exit` (bars), `fx{}`, `type:"modulator"`.
Modulations: `{type:"note|velocity|mute|position", source, target}` (indices into
tracks — re-check them after inserting a track).

## Music-theory cheatsheet (what HVOB / deep-melodic actually use)

- **Dorian mode** is the signature of melodic/deep (its **major IV** gives the
  "lift"). **Phrygian** for dark/tense techno. Minor keys throughout.
- **m7 / m9 / maj7** chords = the jazzy deep-house sound. Simple bass (pedal or
  stepwise). Tension via sus2/sus4 + delayed thirds. Arpeggiate chords for movement.
- **MINIMALISM IS PRODUCTION-CORRECT, not laziness** (Bernard's standing taste):
  "remove 50% of the elements"; one well-designed sound beats stacking five; the
  space between sounds makes the beat hit harder. A big soundbank is a PALETTE —
  each song uses FEW curated voices. Energy = contrast (intro→build→drop→breakdown,
  via `enter`/`exit`), not density.
- Courses: Ableton "Learning Music" (free), Berklee "Composing & Producing EM 1"
  (free audit), Attack Magazine "Deep House Chords", PML "Music Theory for Melodic
  Techno and House".

## Verification discipline (Art. 2 for audio)

The spectral metric (FFT low/mid, RMS, onset count, energy profile) is a PROXY —
it under-reports (pads in low-mids read as "bass-heavy") and only reliably catches
DEAD/FLAT output. Always: (1) programmatic ALIVE check (non-silent, no NaN,
dynamic-not-flat), then (2) `afplay`/`open` the artifact for the human's ear — the
ground truth. Never claim "sounds good" from the metric. See the playbook rules
`verify-before-assuming` (Rule 17) and `codex-orchestration` (batch fan-out).

## Voice layer (Korda's spoken-vocal signature)

Recorded with `ffmpeg -f avfoundation` (the operator records the mic; the user only
speaks). Trimmed/normalized/faded into `samples/vozN.wav`, triggered as a
`sample:<name>` Track (inherits polymeter + pitch via `note`). Make a voice
unrecognizable with `fx: {reverse, pitch, granular:{grain_ms,scatter}}`.

## mp3 note

pydub is avoided (it needs stdlib `audioop`, removed in Python 3.13+). WAV via
stdlib `wave`, mp3 via an `ffmpeg` subprocess.
